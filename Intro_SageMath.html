¿Qué es SageMath?¶

SageMath es un sistema de lgebra computacional nacido en 2005 de la mano del matemtico estadounidense William A. Stein. Inicialmente se llam SAGE, acrnimo de System for Algebra and Geometry Experimentation, denominacin cambiada posteriormente para evitar confusiones con otros programas.

SageMath fue concebido con el propsito de construir una alternativa de software libre a programas comerciales como Mathematica, Matlab, Maple o Magma. Todos estos programas proporcionan, en mayor o menor medida, un entorno en el que desarrollar, entre otras, tareas de clculo simblico, clculo numrico, anlisis estadstico, programacin o representacin grfica de funciones y datos. Son, por ello, programas muy usados y apreciados en los mbitos cientficos y de ingeniera. Presentan, sin embargo, varios inconvenientes. Por un lado, las licencias de uso suelen tener un precio elevado, poco asequible fuera de los mbitos empresariales o acadmicos. Por otra parte, cada programa tiene su propio lenguaje, que es preciso aprender. Pero, sobre todo, se trata de programas de cdigo cerrado: es imposible acceder a la implementacin de las funciones y rutinas que ofrecen, lo cual dificulta notablemente la verificacin de resultados o la correccin de los errores que todo programa inevitablemente conlleva.

SageMath sigue un modelo distinto, bien representado por esta ilustracin, obra de Martin Albrecht:

En lugar de desarrollar por entero cada parte del sistema, SageMath amalgama ms de 100 paquetes de software matemtico (Maxima, GAP, Pari/GP, Numpy, Scipy...), todos de cdigo abierto, a los que se aaden multitud de mdulos escritos en Python para proporcionar una interfaz comn a todos los programas e incrementar la funcionalidad de todo el sistema. Como expresa uno de sus lemas, SageMath no reinventa la rueda, sino que aprovecha e integra software libre ya existente. Se construye as un potente automvil computacional con mltiples beneficios para el usuario. En primer lugar, no es necesario aprender el lenguaje especfico de cada programa, sino que basta con usar Python para comunicarse con todos ellos y trasvasar resultados de uno a otro, todo ello, adems, de forma transparente. SageMath es gratuito, se puede descargar y usar libremente. Por ltimo, SageMath es un sistema de cdigo abierto: cualquiera puede examinar el cdigo, extenderlo, proponer mejoras o corregir errores.

Acceso a SageMath¶

Tres son las vas principales de acceso a SageMath:

  • SageMathCell: Es una pgina web facilitada por una red de servidores distribuidos por todo el mundo. Presenta un cuadro en el que introducir y ejecutar cdigo escrito en SageMath (y tambin en Python y otros lenguajes). Cuenta con una API para integrar SageMathCell en cualquier pgina web.

  • CoCalc: Es un servicio en lnea que facilita un entorno completo de computacin en el que se incluye SageMath. Ofrece cuentas gratuitas, con capacidades suficientes para empezar a experimentar, y cuentas de pago para un uso ms serio o profesional.

  • Instalacin local: SageMath es un software totalmente gratuito. Se obtiene mediante la pgina de descargas de su sitio web oficial (cf. http://www.sagemath.org). Est disponible para varias plataformas, entre ellas, las tres principales (Windows, macOs y Linux).

As pues, para utilizar SageMath basta con disponer de un navegador y una conexin a internet. SageMathCell es apta para ejecutar programas que requieran de pocas lneas de cdigo y escaso tiempo de ejecucin. Para tareas ms avanzadas es mucho ms conveniente registrarse en CoCalc. Si es posible, no obstante, la mejor opcin es contar con una instalacin local.

Tanto en Cocalc como en una instalacin local, son dos los modos fundamentales de uso de SageMath:

  • mediante una terminal, trabajando, como en Python, de forma interactiva;

SageMath y Python¶

El lenguaje usado en SageMath es, en esencia, el lenguaje Python enriquecido con ms funciones, clases y algn convenio de escritura adicional. Hasta la versin 8.9, SageMath se ha basado en Python 2. Desde la aparicin de la versin 9.0 en enero de 2020, la distribucin oficial de SageMath utiliza Python 3.

Dos son las diferencias de sintaxis ms significativas entre Python y SageMath. En Python, el operador ^ es una disyuncin exclusiva (operacin XOR) realizada bit a bit; en SageMath, en cambio, se puede usar ^, adems del operador **, para indicar potenciacin. Del mismo modo, en Python, la expresin m/n, con m y n enteros, devuelve el resultado de la divisin, dando lugar a un nmero de tipo float. Clases y métodos¶

SageMath, al igual que Python, hace uso de la denominada programacin orientada a objetos. En este tipo de programacin se considera que cada elemento que manipula un programa es un objeto que pertenece a una clase determinada. La clase define la estructura de datos del objeto y las operaciones, llamadas mtodos, que se pueden efectuar sobre l. Para aplicar un mtodo mmm sobre un objeto ooo se escribe ooo.mmm(). def primos_hasta(num, verboso=False):
    """
    Obtencin de todos los nmeros primos menores que un nmero num
    mediante la criba de Eratstenes.
    Entrada: num (nmero entero positivo) y verboso (variable booleana). 
             Si verboso=True, se imprimen los clculos intermedios.
    Salida: Lista de los nmeros primos menores que num.
    """
    num_inicial = num
    if verboso and num%5!=0:
        # Ajuste de num al siguiente mltiplo de 5
        # para facilitar la impresin de resultados
        num = 5 * (num//5+1)
    es_primo = num*[True]
    es_primo[0] = es_primo[1] = False
    j = 2
    while j**2 <= num:
        if es_primo[j]:
            for k in range(j**2,num,j):
                es_primo[k] = False
            if verboso:
                print("\nIteracin j=", j)
                cuadro = [[ [5*i+k, es_primo[5*i+k]] for k in range(5)] 
                          for i in range(num//5)]
                print(table(cuadro))
        j += 1
    return [j for j in range(num_inicial) if es_primo[j]]

Es posible que a lo largo de una sesin con SageMath se acabe perdiendo el significado de alguna variable o funcin predefinida. Por ejemplo, tras las asignaciones i = 5 y cos = [1, 2, 3], la variable i deja de representar la unidad imaginaria, mientras que cos ya no es la funcin coseno, sino una variable que almacena una lista.

La orden restore() recupera el valor original de las variables y funciones predefinidas. La orden reset() es ms radical, pues adems borra todas las variables y definiciones introducidas por el usuario. Si slo se quieren restaurar unas variables o funciones concretas, digamos aaa, bbb y ccc, se emplea la orden reset("aaa, bbb, ccc"). Manipulación simbólica y algebraica¶

Expresiones simbólicas¶

Decamos anteriormente que SageMath es capaz de manipular nmeros de forma simblica al objeto de realizar clculos de forma exacta. Veamos, de hecho, qu tipo de dato asocia SageMath a pi o sqrt(3):

In [31]:
print(type(sqrt(3)))
print(type(pi))

<class 'sage.symbolic.expression.Expression'>
<class 'sage.symbolic.expression.Expression'>

As pues, tanto pi como sqrt(3) pertenecen a la clase de expresiones simblicas, representada por SR (del ingls Symbolic expression Ring). A esta clase tambin pertenecen todas las expresiones que dependan de una o ms indeterminadas o variables simblicas. La letra x es, por omisin, una de estas variables. Si se necesitan ms, hay que declararlas con la orden var. Los nombres de las variables simblicas han de ser expresiones vlidas como variables de Python (son legales mivar, miVar, mi_var, mi2var, etc.; no lo es 2mivar).

Pongamos algunos ejemplos. La celda siguiente muestra cmo factorizar un polinomio y activa las salidas con LaTeX, que sern ahora ms adecuadas:

In [32]:

Out[32]:

Mostramos a continuacin cmo declarar un par de variables simblicas nuevas y cmo expandir una expresin algebraica:

In [33]:
var("y,z")
expand((y-z)^4)   # o bien ((y-z)^4).expand()
Out[33]:

Una vez declarada una variable, se puede seguir utilizando en las celdas posteriores. Descomponemos ahora el cociente $\frac{y^2 - y + 4}{y^2 - 1}$ en fracciones simples :

In [34]:

p.partial_fraction()
Out[34]:

Podemos usar las expresiones simblicas para verificar identidades matemticas, como $$\cos 3 z= \cos^3 z-3 \cos z \operatorname{sen}^2 z.$$ Podemos comprobarlo de este modo:

In [35]:

In [36]:

Sustituciones¶

No se pueden hacer asignaciones a las variables simblicas. Si se escribiese x = 10, por ejemplo, x dejara de ser una variable simblica y pasara a ser una variable Python normal que almacena el valor 10. Si se quisiera volver a usar x como variable simblica, habra que declararla de nuevo con var o recuperarla con restore. Qu hacer entonces para hallar el valor de una expresin simblica cuando las variables que en ella intervienen han de tomar valores concretos? Lo que se hace es una sustitucin, pasando los valores de forma adecuada.

Pongamos un ejemplo concreto. Consideremos la siguiente expresin simblica:

In [39]:
var("x,y,z")
f = x + y^2 + z^3

Supongamos que queremos evaluar f cuando x, y y z toman los valores 1, 2 y 3, respectivamente. Cada una de las siguientes celdas contiene un par de modos lcitos de hacerlo:

In [40]:
print(f.subs(x=1, y=2, z=3), f(x=1, y=2, z=3))

32 32
In [41]:
print(f.subs({x:1, y:2, z:3}), f({x:1, y:2, z:3}))

32 32
In [42]:
valores = dict(x=1, y=2, z=3)
print(f.subs(**valores), f(**valores))

32 32
In [43]:
valores = {x:1, y:2, z:3}
print(f(valores), f.subs(valores))

32 32

Funciones simbólicas¶

Las funciones definidas con la sintaxis de Python se pueden evaluar en variables o expresiones simblicas, dando lugar a expresiones simblicas que se pueden manipular como cualquier otra. Por ejemplo, en la siguiente celda se define una funcin $f$ con la orden def:

In [44]:
def f(t):
    return (2*t^2-t+4) / (3*t^2-1)

Observemos que $f$ es una mera funcin Python:

In [45]:
type(f)
Out[45]:

Pero, como x es una variable simblica, f(x) es una expresin simblica. Aprovechmoslo, por ejemplo, para calcular la derivada de $f$:

In [46]:
derivative(f(x),x).rational_simplify()
Out[46]:

Y tambin el lmite de $f$ cuando $x\to+\infty$:

In [47]:
limit(f(x),x=+oo)
Out[47]:

Hipótesis¶

En ocasiones SageMath no transforma o simplifica una determinada expresin porque carece de la informacin necesaria para hacerlo:

In [57]:
sqrt(x^2)
Out[57]:

Si precisamos que x representa un nmero real positivo o negativo, entonces SageMath realiza la simplificacin correcta:

In [58]:
forget(); assume(x>0)
print("sqrt(x^2)=", sqrt(x^2), "si x>0")
forget(); assume(x<0)
print("sqrt(x^2)=", sqrt(x^2), "si x<0") 
forget()

sqrt(x^2)= x si x>0
sqrt(x^2)= -x si x<0

Con forget se borra cualquier hiptesis previa que se haya podido realizar (es innecesario si no se ha efectuado ninguna). Con assume se le indica a SageMath que manipule las expresiones siguientes de acuerdo con la hiptesis que figura en su argumento.

Ecuaciones y sistemas no lineales¶

SageMath dispone de la orden solve para resolver algebraicamente ecuaciones y sistemas no lineales. Hallemos, por ejemplo, las races del polinomio $p(x)=x^6 - 9x^5 + 12x^4 + 76x^3 - 144x^2 - 192x + 256$:

In [59]:
p = x^6 - 9*x^5 + 12*x^4 + 76*x^3 - 144*x^2 - 192*x + 256
solve(p==0, x)
Out[59]:

Cuando el segundo miembro de la ecuacin es $0$, se puede omitir ste. Luego hubiera bastado con solve(p, x) para obtener el mismo resultado. Observemos, por otra parte, que, como $p$ es un polinomio de grado $6$, deberamos haber obtenido $6$ races. Lo que sucede es que algunas races son mltiples. Para saber la multiplicidad de cada una se usa el argumento opcional multiplicities:

In [60]:
solve(p==0, x, multiplicities=True)
Out[60]:

Queda ahora claro que $1$ es una raz simple de $p$, $-2$ es una raz doble y $4$ es una raz triple. Si se prefiere expresar las soluciones mediante diccionarios, se aade el argumento solution_dict:

In [61]:
solve(p==0, x, solution_dict=True)
Out[61]:

Una vez resuelta una ecuacin, lo habitual es utilizar el valor de una o ms soluciones. Para extraer este valor, lo ms simple es usar el mtodo subs:

In [62]:
soluciones = solve(p==0, x)   
x.subs(soluciones[1])  # extraccin del valor de x de la segunda solucin
Out[62]:
In [63]:
soluciones = solve(p==0, x)
raices = [x.subs(sol) for sol in soluciones]
raices   # lista de todas las races de p
Out[63]:

En la medida de lo posible, solve tiene en cuenta las hiptesis que hayan podido hacerse:

In [64]:
forget()             # borrado preventivo de toda hiptesis
assume(x>0)          # imponemos la restriccin x>0
show(solve(p==0, x)) # slo se devuelven las races positivas de p
forget()             # anulamos la hiptesis para que no influya en otros clculos

La orden solve puede trabajar con ecuaciones puramente simblicas:

In [65]:
var("a,b,c")
solve(a*x^2 + b*x + c == 0, x)
Out[65]:

Veamos ahora algunos ejemplos de sistemas de ecuaciones. Hallemos los puntos de interseccin de la circunferencia $x^2+y^2=5$ y la recta $y=2x-1$:

In [66]:
solve([x^2+y^2==5, y==2*x-1], x, y)
Out[66]:

Vemos que hay dos puntos de interseccin. En este caso las soluciones se expresan de forma exacta. No siempre es posible. Con frecuencia hay que contentarse con una aproximacin numrica, como muestra el resultado de la siguiente celda, en la que se calculan los puntos de corte de la hiprbola $x^2-3x-y^2=1$ y la circunfencia $x^2-2x+y^2-y=6$:

In [67]:
solve([x^2-3*x-y^2==1, x^2-2*x+y^2-y==6], x, y)
Out[67]:

La siguiente celda presenta las soluciones de una forma seguramente ms clara:

In [68]:
soluciones = solve([x^2-3*x-y^2==1, x^2-2*x+y^2-y==6], x, y)
puntos = [(x.subs(sol),y.subs(sol)) for sol in soluciones]
for punto in puntos:
    show(punto)

En el cdigo precedente, para evitar la repeticin de subs, se puede crear primero un vector de componentes x e y y luego efectuar la sustitucin:

soluciones = solve([x^2-3*x-y^2==1, x^2-2*x+y^2-y==6], x, y)

    show(punto)
